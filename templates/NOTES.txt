MODE: Full (RAG + document connectors enabled)

IMPORTANT — Vespa requires the "anyuid" SCC:
  Vespa's container image needs to run as a specific user which conflicts
  with OpenShift's default restricted SCC. Ask your cluster admin to run:

    oc adm policy add-scc-to-user anyuid \
      -z {{ include "onyx.fullname" . }}-vespa \
      -n {{ .Release.Namespace }}

  This grants anyuid ONLY to the Vespa ServiceAccount in your namespace.
  No cluster-wide privileges are needed. After granting:

    kubectl delete pod -l app.kubernetes.io/component=vespa -n {{ .Release.Namespace }}

  The pod will restart with the correct SCC.
